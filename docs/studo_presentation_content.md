## Slide 2: The Core Vision - What is Studo?

Studo is a groundbreaking platform designed to provide a **free and open-source spatial computing environment** for education and research. It allows users to interact with complex 3D models, perform scientific experiments, and access a vast knowledge base through intuitive gestures and AI-powered assistance.

*   **Key Features:** Recursive 3D model exploration, AI-driven analysis, gesture control, multi-device accessibility.

## Slide 3: The Problem We Solve

Traditional learning methods often lack interactivity and accessibility, especially for complex scientific and engineering concepts. Expensive software and hardware create barriers for students, particularly in underserved communities. Studo aims to break these barriers by offering a powerful, free, and universally accessible learning tool.

*   **Barriers:** Cost, complexity, lack of engagement.
*   **Studo's Solution:** Free, intuitive, immersive, and universally accessible.

## Slide 5: Frontend: The Immersive Experience

*   **Technology:** React, Vite, Three.js (R3F), MediaPipe, Tailwind CSS.
*   **Functionality:**
    *   **3D Holographic UI:** Glassmorphic, neon-accented interface.
    *   **Recursive 3D Models:** Explore complex structures with deep zoom capabilities (e.g., human body, car engine).
    *   **Gesture Control:** Natural hand movements (pinch, rotate, explode) for model manipulation via MediaPipe.
    *   **Multi-Device:** Runs on Laptops (Windows/Mac/Linux) and Mobile (Android/iOS) as a Progressive Web App (PWA).

## Slide 6: Backend: The Intelligent Core

*   **Technology:** Python (FastAPI), Socket.io.
*   **Functionality:**
    *   **Real-time Communication:** Bidirectional data flow between frontend and backend for seamless interaction.
    *   **AI Brain Orchestration:** Manages requests to the local AI for analysis and knowledge retrieval.
    *   **Simulation Engine:** Processes physics and mathematical calculations for experimental outcomes.
    *   **Data Scrapers:** Gathers information from scientific databases and research papers.

## Slide 7: The Local AI Brain (ARIA) - No API Keys, No Limits

*   **Technology:** Ollama + Llama 3 (or Mistral).
*   **Functionality:**
    *   **Contextual Understanding:** Processes user queries with deep contextual awareness of the displayed 3D model.
    *   **Scientific Analysis:** Analyzes experimental changes based on physics, chemistry, and biology principles.
    *   **Knowledge Retrieval:** Utilizes web scraping (DuckDuckGo Search API, ArXiv) to access and summarize vast scientific literature.
    *   **Voice Interaction:** Powers natural language understanding and generation for voice commands and responses.

## Slide 8: 3D Model & Image Generation: The "Murtikar" Approach

*   **Technology:** Stable Diffusion XL (SDXL), Gaussian Splatting/Zero-1-to-3.
*   **Functionality:**
    *   **High-Fidelity 3D Generation:** Creates detailed 3D models from textual descriptions or 2D images.
    *   **Dynamic Model Creation:** Generates missing components or variations for experimental scenarios.
    *   **Visual Learning:** Provides rich, accurate visual representations for all scientific concepts.

## Slide 9: Audio & Voice: Natural Interaction

*   **Technology:** Piper TTS (Text-to-Speech), Web Speech API (Speech Recognition).
*   **Functionality:**
    *   **Natural Voice Output:** Converts AI responses into human-like speech for an immersive experience.
    *   **Accurate Voice Input:** Allows users to control Studo and query ARIA using natural voice commands.
    *   **Offline Capability:** Piper TTS can run locally, ensuring functionality even without internet access.

## Slide 10: Multi-Device & Accessibility

*   **PWA (Progressive Web App):** Single codebase for all platforms (Desktop & Mobile).
*   **Docker/Docker Compose:** One-command setup for the entire system, making it easy for anyone to run.
*   **Lightweight Frontend:** Ensures smooth performance even on low-end devices.
*   **Free & Open-Source:** No subscription fees, no API costs, truly accessible to all.

## Slide 11: Security, Debugging & Future

*   **Security:** Local-first data, CORS, Rate Limiting.
*   **Debugging:** Sentry for error tracking, browser dev tools.
*   **MCP (Model Context Protocol):** Custom MCP for AI tool orchestration.
*   **Future:** Collaborative multi-user labs, advanced physics simulations, integration with educational platforms.
